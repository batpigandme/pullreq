Scenario

Fixing documentation (e.g. typos) seems to me to be the lowest hanging fruit/most likely scenario for someone’s first PR in the tidyverse.

Outline

This PR flow starts with the following four steps:

  1. Fork
  2. Clone
  3. R CMD check (devtools::check())
  4. Branch

Fork & Clone
